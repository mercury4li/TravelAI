# 旅行规划助手 — 竞品调研报告（聚焦版）

> **文档日期**：2026-05-01  
> **调研范围**：圆周旅迹（常被口语称为「圆周轨迹」）、豆包、Mindtrip、Travo、Plantrip。  
> **说明**：功能与定价以各产品 **官网 / App 商店 / 应用内** 为准；用户评论为 **公开评价聚合**，存在时间与样本偏差。下文「圆周旅迹」即市场常见同名 App。

---

## 1. 调研目的

为自研「旅行规划助手」聚焦直接竞品：**国内垂直工具 + 通用大模型 + 海外 AI 规划第一梯队**，便于对齐功能模块、商业形态、用户口碑与差异化切口。

## 3. 圆周旅迹

### 3.1 主要功能模块

- **一键导入**：小红书、微信公众号、大众点评等链接 / 图文，解析地点与时间，生成行程骨架。  
- **智能规划 / 地图**：按天展示路线、距离与驾车时间；天气与交通信息辅助；地图总览与分享（海报、长图等）。  
- **多人协作**：邀请好友共同编辑、口令分享。  
- **附近推荐**：按停留点推荐周边小店与打卡点。  
- **辅助**：行程清单、预算统计（能力深度随版本变化，以 App 为准）、足迹类记录等。

### 3.2 优势

- **种草 → 结构化行程**路径短，符合国内内容习惯。  
- **地图可视化 + 协作**在同赛道中清晰。  
- 多数用户认为 **界面相对干净**、无重度内购打扰（以当时版本商店描述为准）。

### 3.3 劣势

- **自动动线质量不稳定**，常需大量手改；更像「可视化编排」而非最优求解器。  
- **机酒门票闭环弱**，需跳转 OTA。  
- **强依赖联网**，弱网/机上体验差。  
- **预算/记账/行李**等全流程工具不如综合 OTA 或专项 App 完整。  
- **解析依赖原文质量**，错漏需用户自行纠错；公开数据少披露解析成功率。  
- 营销曝光多，部分用户对 **长期免费与数据信任** 存疑。

### 3.4 用户故事（示例）

1. **作为** 小红书重度用户，**我希望** 复制笔记链接就得到按天地图展示的行程，**以便** 少做表格、直接发给同伴改。
2. **作为** 与朋友自由行的人，**我希望** 多人同时改同一行程，**以便** 不用在微信群反复发版本。
3. **作为** 已订好机酒的人，**我希望** 把攻略里的景点排进已有日期，**以便** 和真实预订对齐（若产品支持手动对齐则满足，否则仍痛苦）。

### 3.5 定价与付费方式

- App Store 等渠道显示 **免费下载**；检索未见到权威、统一的 **会员 / 内购** 公开说明。  
- **付费方式**：若有内购/订阅，以 **App 内购买与商店页** 为准；当前调研结论：**偏免费工具型**，商业化形态需持续跟踪版本更新。

### 3.6 用户评论中的优点 / 缺点（归纳）


| 优点（常见）          | 缺点（常见）             |
| --------------- | ------------------ |
| 一键导入小红书等省时；地图直观 | 自动规划「不够聪明」、要手改     |
| 协作与分享方便         | 必须联网；飞机上难用         |
| 界面简洁、广告感相对低     | 改版后部分入口变深（如收藏）     |
| 导出/分享行程单受好评     | 缺少记账、清单等；时间选择控件不顺手 |


**来源类型**：App Store / 应用商店评论、垂直媒体介绍、[V2EX 等社区](https://www.v2ex.com/t/1162441)。

## 4. 豆包

### 4.1 主要功能模块（与「旅行规划」相关部分）

- **多轮对话**：自然语言描述目的地、天数、预算、人群，生成文字版日程与建议。  
- **联网搜索 / 插件能力**（以当前客户端为准）：可补充时效信息，但质量依赖检索结果与模型整合。  
- **语音、图像、文档**：上传攻略图、行程表等辅助理解（具体能力以产品版本为准）。  
- **非旅行专用**：无独立「行程对象」、地图协作、订单邮箱解析等原生旅行数据模型。

### 4.2 优势

- **零安装门槛**（若已用字节系账号）、**响应快**、适合 **头脑风暴与多版本草案**。  
- **中文表达与本土语境**通常优于纯海外工具。  
- **泛场景**：同一助手可问签证、装备、文案等，不仅限于行程。

### 4.3 劣势

- **事实性错误**：车次不在时刻表、车站混淆、景点时长与体量不匹配等均有公开测评案例。  
- **时效与可订状态**：闭园、限流、库存难保证；**必须人工核对**官方渠道。  
- **个性化易表面化**：不同约束下输出仍可能雷同；细粒度如推车、午休、无障碍覆盖不足。  
- **无结构化行程资产**：版本管理、多人协作地图、与 OTA 实价联动弱。  
- **价格与库存**：对话中的报价未必对齐任一平台。

### 4.4 用户故事（示例）

1. **作为** 不想装新 App 的人，**我希望** 用几句话生成一份「大阪 5 日」草案，**以便** 再复制到备忘录或发给朋友。
2. **作为** 行前焦虑型用户，**我希望** 不断追问「带老人怎么改」，**以便** 快速迭代文案。
3. **作为** 严谨出行者，**我希望** 输出能直接执行，**以便** 不二次查表——**当前产品难以满足**，需自行校验。

### 4.5 定价与付费方式

- **C 端**：豆包作为大众对话产品，**以免费使用为主**；是否存在会员、增值包、更高模型档位等，**以字节跳动官方说明与 App 内标价为准**（第三方 SEO 报价差异大，本报告不采信单一非官方数字）。  
- **B 端 / API**：大模型按 **Token 调用计费** 等模式与官网控制台为准。

### 4.6 用户评论中的优点 / 缺点（归纳）


| 优点（常见）               | 缺点（常见）         |
| -------------------- | -------------- |
| 生成快、表达流畅、改 prompt 方便 | 交通/开放时间易错，不能照抄 |
| 中文攻略结构完整、适合打底稿       | 餐饮重复、时段安排不合理   |
| 可结合搜索（视版本）           | 与真实订票价格脱节；协作弱  |


**来源类型**：用户自测博文、媒体「旅行 Agent」对比稿、行业稿对 AI 旅行的风险提示。

## 5. Mindtrip

### 5.1 主要功能模块

- **对话式规划 + 实时地图可视化**：推荐落在地图与地点卡片（含图、评、天气等）。  
- **Start Anywhere®**：粘贴链接、视频、截图、PDF 等，抽取地点并生成列表或路线。  
- **Collections / Google Pins 导入**：主题收藏与从 Google Maps 收藏导入。  
- **灵感页 Inspiration**：浏览他人行程，分叉再编辑。  
- **多人协作**：行程内群聊、@Mindtrip 协调偏好。  
- **收据与凭证**：上传或转发至 `receipts@mindtrip.ai` 集中管理。  
- **Events**：附近活动与票务线索。  
- **预订入口**：酒店、航班、餐厅、体验等；**租车、Tours** 等在官网标为 **Coming soon**。  
- **创作者与 Mindtrip for Business**：B 端与目的地营销。

### 5.2 优势

- **「能看、能点、能订一部分」** 的体验链完整度高于纯聊天工具。  
- **Start Anywhere** 与 **社区真人指南** 增强信任（E-E-A-T）。  
- 独立评测中：**闭馆日、潮汐** 等约束有正面案例；**途中临时追问**（如「某城 1.5 小时怎么玩」）表现突出。  
- **酒店等可展示多 OTA 报价入口**（评测描述），强于「只给文字」类工具。

### 5.3 劣势

- **门票类价格** 可出现 **估算高于官网** 的情况（如迪士尼门票区间），预算需回官网核对。  
- **景点门票等多数不能应用内一键买完**，与更偏「代理闭环」的产品相比有断点。  
- **与真实世界仍不同步**：闭馆日、边境政策、大型活动等，与其它 AI 一样存在盲区（实测文共识）。  
- **极复杂多段机票** 场景，有评测认为竞品更顺手（主观参考）。  
- **租车等模块未上线**，「全品类一站式」叙事未闭合。  
- **国内用户**：国际 OTA 与账号习惯是否顺滑需单独实测。

### 5.4 用户故事（示例）

1. **作为** 看 TikTok / YouTube 种草的人，**我希望** 贴链接就得到可编辑地图行程，**以便** 少自己扒地点。
2. **作为** 家庭游规划者，**我希望** AI 记得孩子年龄并避开闭馆日，**以便** 少踩雷。
3. **作为** 想直接订酒店的人，**我希望** 点卡片就看到实价并跳转，**以便** 少开十个浏览器标签。
4. **作为** 乐园深度玩家，**我希望** 门票价格在应用内就是最终价，**以便** 一次完成预算——**当前仍有缺口**。

### 5.5 定价与付费方式

- 多篇独立评测写 **个人旅行规划侧为免费使用**；**企业 / 目的地 / 酒店** 合作形态为 **商务报价**（官网 Mindtrip for Business）。  
- **付费方式**：C 端可能通过 **预订佣金、B 端订阅** 等变现，以官方条款为准；本报告不臆测未公开的 C 端订阅价。

### 5.6 用户评论 / 第三方评测中的优点 / 缺点（归纳）


| 优点（常见）              | 缺点（常见）                  |
| ------------------- | ----------------------- |
| 地图 + 卡片信息量大、设计感好    | 门票价格偏估、需官网复核            |
| Start Anywhere 差异化强 | 不能在一个应用里订齐所有票种          |
| 协作与灵感页适合结伴出行        | 部分品类「Coming soon」，闭环未齐  |
| 路上临时追问体验好（实测）       | 与 Layla 等「更重预订」产品比，代理感弱 |


**来源类型**：[mindtrip.ai](https://www.mindtrip.ai/)、[AITravel.tools Mindtrip 评测](https://aitravel.tools/mindtrip-review/)、[米兰科莫湖实测对比](https://aitravel.tools/best-ai-trip-planner-milan-lake-como-test/)。

## 6. Travo

### 6.1 主要功能模块

- **AI 行程生成**：按兴趣与日期生成景点与日程；**动线优化**（减少折返）。  
- **行程收纳**：航班、酒店、租车、活动、**多币种支出**。  
- **社交链接导入**：Instagram / TikTok / YouTube 链接识别地点并加入行程。  
- **Quick Escape**：基于所在城市的 **周末短途** 一键方案，可 shuffle 目的地。  
- **邮件自动导入**（Premium）：转发确认邮件抽取预订写入行程。  
- **音频故事 / 导览**：多语言音频内容（官网称 13 种语言等）。  
- **离线**：Premium 强调地图、行程、音频离线可用。  
- **分享**：与同行者共享计划。

### 6.2 优势

- **规划 + 收纳 + 路上体验（音频）+ 离线** 组合在同质化市场里较清晰。  
- **邮件导入** 与 **社交抓点** 降低录入成本。  
- **订阅价相对低**（见下表），适合常旅客试用 Premium。

### 6.3 劣势

- **独立「真上路」多工具横评**中曝光度不如 Mindtrip / ChatGPT 等（样本问题），**第三方口碑密度**需自行积累调研。  
- **AI 共性**：营业时刻、临时事件等与真实世界仍有偏差。  
- **免费档限制紧**：行程数与 AI 次数少，深度用户较快触墙（以 [travo.me](https://travo.me/) FAQ 为准）。  
- **社交源** 偏 Ins/TikTok/YouTube，与国内 **小红书主链路** 的重叠度需对比圆周旅迹评估。

### 6.4 用户故事（示例）

1. **作为** 已订好机酒的人，**我希望** 转发邮件后行程自动出现航班酒店，**以便** 再让 AI 补景点动线。
2. **作为** 国际漫游贵的人，**我希望** 离线仍能看地图与行程，**以便** 在当地不迷路。
3. **作为** 周末想说走就走的人，**我希望** 输入城市就得到周五–周日草案，**以便** 快速决策目的地。

### 6.5 定价与付费方式


| 档位          | 内容摘要（以官网 / 商店为准）                                                                     |
| ----------- | ------------------------------------------------------------------------------------ |
| **免费**      | 约 **2 个完整行程**、约 **2 次 AI 行程生成**；地图探索等有每日限制（见 [travo.me](https://travo.me/) FAQ）。     |
| **Premium** | **无限**行程与 AI 生成、完整离线、无广告、优先支持等；约 **$3.09/月** 或 **$25.99/年**；含试用策略（以 App Store 结算为准）。 |


**付费方式**：**App 内订阅**（iOS / Google Play）。

### 6.6 用户评论 / 公开材料中的优点 / 缺点（归纳）


| 优点（常见）               | 缺点（常见）          |
| -------------------- | --------------- |
| 一站式收纳 + AI 动线        | 免费次数少，很快要订阅     |
| 离线、音频适合落地游玩          | 对「仅想要免费草稿」用户门槛高 |
| 周末 Quick Escape 场景清晰 | 中文环境与国内种草源适配需自测 |

## 7. Plantrip

### 7.1 主要功能模块

- **AI 行程生成**：秒级按天输出；可选 **上午/下午/晚间** 或 **按小时** 粒度。  
- **导出与分享**：**PDF**、邮件、链接复制、社交分享等。  
- **行李清单**：按场景生成打包清单（官网宣称大量清单生成量）。  
- **预算 / 天气 / 旅行贴士**：与行程打包呈现（以生成结果为准）。  
- **地图集成**（定价页写明）。  
- **开发者入口**：站点提供 Developer 相关入口，是否开放完整 API 以官方文档为准。

### 7.2 优势

- **出稿极快 + 可交付**（PDF），适合 **给同事/客户/家人一版可读方案**。  
- **粒度可选**，兼顾扫读与细排。  
- **免费档可月度试用数次**，门槛低（见定价页）。

### 7.3 劣势

- **预订与履约闭环弱**：以「文案 + 导出」为主，**库存与出票**不在同一产品内完成。  
- **企业级集成**：第三方评测认为 **集成面偏窄**，不适合指望 CRM/预订引擎全自动打通的团队（需搭配其他系统）。  
- **公开评价基数小**：Trustpilot 等 **评论量少**，大规模口碑难对标头部产品。  
- **移动端**：Web 为主力；Android 应用存在但需评估更新频率与体验。  
- **AI 共性**：开放时段、路况等仍需人工校验。

### 7.4 用户故事（示例）

1. **作为** 定制师，**我希望** 10 分钟内出带天与预算说明的 PDF，**以便** 发给客户确认。
2. **作为** 自由行用户，**我希望** 先按半日块看骨架再展开按小时，**以便** 控制规划节奏。
3. **作为** 不想装 App 的人，**我希望** 浏览器里完成并下载 PDF，**以便** 存邮件附件。
4. **作为** 要在应用内直接锁票的人，**我希望** 点即订——**当前产品不主打此路径**。

### 7.5 定价与付费方式

| **Plus**  | **$3/月**（年付约合 **$2.5/月**、年账单 **$30**）；无限 Step-by-Step、个性化行李清单、更丰富酒店/交通/旅行要求等字段、**私密行程**、去广告。 |


**付费方式**：**信用卡等在线订阅**（USD 标价，结账时可能显示本地货币）。

## 8. 结论与对自研产品的启示

1. **「导入」赛道**：圆周旅迹已占 **小红书等国内链路**；若跟进，需在 **动线质量、可验证数据、离线** 上拉开差距。
2. **「对话」赛道**：豆包证明 **需求存在**，但 **结构化资产 + 校验工作流** 才是旅行垂直机会。
3. **「可视化 + 部分预订」**：Mindtrip 拉高体验标准；短板在 **门票精度与全闭环**。
4. **「收纳 + 离线 + 音频」**：Travo 差异化清晰；需直面 **免费档限制** 与 **国内内容源** 问题。
5. **「提案 + PDF」**：Plantrip 适合 **B 端提案 / 轻用户**；与「旅行中」场景距离较远。

**下一手建议**：选定 2～3 款做 **同题实测**（同一目的地+同一约束），记录 **错误类型**（时刻表、闭馆、价格、距离），作为自研 PRD 的 **硬指标** 与测试用例来源。
