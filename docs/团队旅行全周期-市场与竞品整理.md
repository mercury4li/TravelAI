# 团队旅行「全周期」市场与竞品整理

> **信息来源**：[下一个万亿风口：为什么没有一款 App 能搞定「团队旅行」这件事？](https://mp.weixin.qq.com/s/_2ceWQBwhOC6GF2V2BM0Kg)（公众号「小理子墨」）  
> **整理日期**：2026-05-08  
> **说明**：本文为对公开文章的**结构化摘录与归纳**，文中产品数据、月活等为原文引用，使用时建议二次核对官方或第三方统计口径。

---

## 1. 问题陈述：团队在旅行里缺什么

典型痛点被归纳为三类场景：

| 阶段 | 现象（原文归纳） |
|------|------------------|
| **行前** | 攻略、订票、酒店、行程分散在多个 App；信息靠微信碎片化同步 |
| **行中** | 多人垫付、无统一记账；集合与变动难协调 |
| **行后** | 照片分散、游记无人写、分账扯皮周期长 |

**核心论断（原文）**：多数产品只覆盖旅行链条中的**某一阶段**，缺少把行前、行中、行后串成「一趟旅程」的**完整容器**；用户更需要连续体验，而非三个割裂的 App。

---

## 2. 框架：旅行的三个阶段

### 2.1 行前规划（竞争最激烈）

原文列出的五个环节：

1. 订票（机票 / 火车 / 船票等，偏流量入口）  
2. 订住宿（酒店 / 民宿 / 青旅）  
3. 行程设计（路线、景点、时间管理）  
4. 资源收集（攻略、POI、他人经验）  
5. **团队决策**（去哪、住哪、怎么去——多人协商痛点集中）

### 2.2 途中管理（原文认为最被忽视、需求密集）

- 记账与分账  
- 实时协调（集合点、行程变更通知）  
- 多设备照片归集  
- 票务 / 订单快速查找  
- 途中临时订餐、改签等支付与预订  

### 2.3 行后沉淀（「回忆经济」）

- 自动生成视频（照片 / 短片合成）  
- 游记（图文记录）  
- 账目清算与分摊结算  
- 向小红书、朋友圈等渠道分发内容

## 3. 竞品与平台全景（按原文分类）

### 3.1 行前 / 综合预订

| 产品 | 原文要点 |
|------|----------|
| **携程 / 飞猪 / 美团 / 同程** | 一站式预订强；携程月活约 1.5 亿、飞猪约 8000 万（原文数据）。弱项：不做完整行程规划、不记账、不生产「行后内容」——偏行前 + 交易。 |
| **穷游行程助手** | 行程编排专业；注册用户约 2388 万，创建行程超 5341 万个。弱项：几乎只做行前，行中、行后薄弱。 |
| **马蜂窝** | 攻略 + 游记社区，月活约 2000 万；攻略质量与 UGC 强；行程工具弱于穷游，商业化路径与「工具闭环」原文认为不够直接。 |
| **小红书** | 月活约 3 亿；攻略鲜活、推荐强。局限：种草与内容为主，非工具 + 非交易闭环。 |

### 3.2 途中：分账 / 记账

| 产品 | 原文要点 |
|------|----------|
| **Splitwise** | 全球团队分账，约 3000 万注册用户；体验与多币种、分摊方式好；只做分账，场景独立。 |
| **Tricount** | 欧洲常用，约 1000 万用户；局限同 Splitwise。 |
| **微信记账本** | 覆盖微信支付用户；自动同步支付记录；偏个人，缺团队分账场景。 |

### 3.3 行后：轨迹 / 视频

| 产品 | 原文要点 |
|------|----------|
| **Polarsteps** | GPS 轨迹 + 时间线 + 按地点归集照片，约 500 万用户；接近「多段覆盖」，但缺记账分账、视频生成与中国市场布局（原文表述）。 |
| **剪映 / CapCut** | 月活量级大；AI 成片、模板强；需手动导入素材，缺「旅行上下文」。 |
| **Quik（GoPro）** | 自动高光剪辑；偏 GoPro 生态。 |

### 3.4 国外「一体化」尝试

| 产品 | 原文要点 |
|------|----------|
| **Wanderlog** | 约 300 万用户；行程 + 预算 + 协作 + 地图，覆盖面广但单项深度有限。 |
| **TripIt** | 约 200 万用户；邮件解析行程、集成度高；偏商务差旅，弱游记 / 视频 / 分账等消费旅行需求。 |
| **Google Trips（已下线）** | 原文作注：曾接近「全栈旅行助手」（Gmail 行程、收藏、离线等），2019 年下线——暗示全栈难度高。 |

## 4. 重点案例：穷游行程助手（原文拆解）

### 4.1 定位与数据（原文）

- 穷游网核心工具，社区沉淀锦囊、游记、攻略。  
- 创建行程数超 **5341 万**，注册用户约 **2388 万**。  

### 4.2 优势（原文）

- 拖拽式日程、路线优化，编排体验好。  
- POI 与全球景点 / 食宿数据积累深，模板库大（可复用他人行程）。  
- 可导出 PDF / Excel 等，偏签证、打印场景。  

### 4.3 短板（原文）

1. **行中几乎空白**——「出发即结束」式假设，与真实旅途需求脱节。  
2. 游记分享流程偏手动，与「AI 生成 + 一键分发」时代体验差距大。  
3. 协作偏「分享链接」，缺群组、分工、实时通知等真协作。  
4. **无记账分账**。  
5. 商业化（商城、保险、搜索排序）干扰找真实内容。  
6. 移动端体验弱于网页，与手机优先习惯不符。  

### 4.4 技术 / 产品层面归纳（原文）

- 缺 AI（行程、游记、视频、账单分析等）。  
- 缺数据闭环（规划期活跃，行中、行后数据回流弱）。  
- 实时协作能力弱。  
- 社交裂变弱，行程多在站内，难对标小红书式外溢。  

**一句话（原文意译）**：长期积累行程与用户，但产品哲学仍停在「把行程排好」，未贯穿花费、影像与情绪体验。

| Splitwise | — | — | — | — | 强 | 无 |
| Polarsteps | — | — | — | — | 无 | 行中 + 游记偏强 |

**结论（原文）**：缺少覆盖 **行前 + 行中 + 行后** 全链条的单一产品。

## 6. 机会方向（原文归纳）

### 6.1 产品逻辑：「旅程容器」

一次旅行 = 一个空间，在内完成：

- **行前**：订单聚合、行程、预算预估  
- **行中**：记账、影像上传、行程微调  
- **行后**：AI 游记、视频成片、自动算账与分账  

### 6.2 三个方向

1. **从行中切入，向上下延伸**  
   - 以**团队记账 + 分账**为高频入口 → 自然扩展照片与行程微调 → 用行中数据驱动行后自动生成（闭环）。  

2. **AI 补齐穷游等传统工具短板**  
   - 自然语言生成行程、照片驱动游记、素材驱动视频、账单与分摊建议、汇率等。  

3. **社交裂变**  
   - 游记 / 视频结尾生成适合小红书、朋友圈的分享卡片，内容外溢带来增长。  

### 6.3 轻启动（原文 MVP 建议）

- 不必一上来全栈。  
- **MVP**：团队旅行记账 + 自动分账，对标 Splitwise，但绑定**旅行场景**（关联行程、照片备注等）。  
- **理由**：频次高、粘性强、结构化数据利于后续延展。  

---

## 7. 结语（与原文主旨对齐）

机会不在于无限加深单一功能，而在于把「**一整趟旅行**」当作统一产品体验：从规划与预订，到途中每一笔消费与影像，再到可分享的回忆与清算——**谁先跑通全链条，谁更可能成长为平台级玩家**。

- 月活、用户数等为**原文引用**，不同统计机构与时间点可能不一致，立项或对外引用前请核实。
